# T-M3-2: エラーハンドリングの実装

| 項目 | 内容 |
|------|------|
| マイルストーン | M3 |
| 依存 | T-M3-1 |
| 要件ID | FR-013 |

## 説明

API エラーやランタイムエラーを分類し、ユーザーに分かりやすいメッセージとして表示するエラーハンドリングを実装する。

## 作業内容

1. CLI のトップレベルで try-catch を実装する
2. API エラーの分類処理を実装する
   - 429: レート制限 → リトライまでの待機時間を表示する
   - 400: リクエスト不正 → パラメータ確認を促す
   - 403: ポリシー違反 → プロンプト内容の見直しを促す
   - 500: サーバーエラー → 時間をおいてリトライを促す
3. ネットワークエラーのハンドリングを実装する
4. タイムアウトエラーのハンドリングを実装する
5. プロセス終了コードを適切に設定する（正常: 0、エラー: 1）

## タスクのチェックリスト

- [x] API エラーが分類されて表示される
- [x] 各エラーに対処方法が含まれている
- [x] 未知のエラーでもクラッシュせず、メッセージが表示される
- [x] エラー時の終了コードが 1 である
