# T-M1-2: 設定管理モジュールの実装

| 項目 | 内容 |
|------|------|
| マイルストーン | M1 |
| 依存 | T-M1-1 |
| 要件ID | FR-011, FR-012 |

## 説明

CLI 引数・環境変数・デフォルト値を統合し、最終的な設定オブジェクトを生成するモジュールを実装する。API キーの存在チェックも行う。

## 作業内容

1. `src/config.ts` を作成する
2. dotenv で `.env` ファイルを読み込む処理を実装する
3. デフォルト値を定義する（model: fast, resolution: 720p, aspect: 16:9, duration: 8）
4. CLI 引数 > `.env` > デフォルト値の優先順位で設定を解決するロジックを実装する
5. `GOOGLE_API_KEY` の存在チェックを実装し、未設定時にエラーメッセージを表示する
6. モデル名のマッピング処理を実装する（fast → veo-3.1-fast-generate-preview, standard → veo-3.1-generate-preview）

## タスクのチェックリスト

- [x] .env から GOOGLE_API_KEY を読み込める
- [x] API キー未設定時に分かりやすいエラーメッセージが出る
- [x] CLI 引数が .env の値を上書きする
- [x] デフォルト値が正しく適用される
- [x] fast / standard がそれぞれ正しいモデル名にマッピングされる
